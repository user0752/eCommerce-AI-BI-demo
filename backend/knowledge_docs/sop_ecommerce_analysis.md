# 电商数据分析标准操作流程 (SOP)

## 1. 数据采集与预处理

### 1.1 数据源确认
- **结构化数据**：MySQL数据库中的ads_region_consume_analysis、ads_product_feature_full、ads_user_item_interaction_matrix、ads_user_profile_full表
- **非结构化数据**：知识文档、市场报告、业务规则说明等

### 1.2 数据清洗
- 处理缺失值：使用均值/中位数填充或删除
- 处理异常值：基于业务规则识别并处理异常值
- 数据标准化：统一数据格式和单位

## 2. 数据分析流程

### 2.1 区域消费分析
1. 确定分析维度：城市、时间、消费类别
2. 选择关键指标：销售额、订单数、客单价、付费率
3. 执行SQL查询获取数据
4. 生成分析报告和可视化图表

### 2.2 商品特征分析
1. 确定分析维度：商品类别、品牌、价格区间
2. 选择关键指标：销量、销售额、转化率
3. 执行SQL查询获取数据
4. 生成热销商品分析报告

### 2.3 用户行为分析
1. 确定分析维度：用户画像、行为类型、时间
2. 选择关键指标：点击量、收藏量、加购量、购买量
3. 执行SQL查询获取数据
4. 生成用户行为分析报告

## 3. 报告生成与解读

### 3.1 报告结构
- 执行摘要：核心发现和建议
- 数据分析：详细数据和图表
- 业务洞察：基于数据的业务建议
- 局限性说明：数据覆盖范围和限制

### 3.2 解读准则
- 以数据为依据，避免主观臆断
- 结合业务上下文进行分析
- 明确说明数据来源和时间范围
- 提供可操作的建议

## 4. 质量控制

### 4.1 数据质量检查
- 检查数据完整性：确保所有必要字段都有值
- 检查数据一致性：确保不同表之间的数据一致
- 检查数据准确性：验证关键指标的计算逻辑

### 4.2 分析质量检查
- 验证分析方法的正确性
- 检查逻辑推理的合理性
- 确保结论与数据一致

## 5. 常见问题处理

### 5.1 数据缺失处理
- 当指定时间范围内无数据时，扩展时间范围
- 当指定城市无数据时，查询全省聚合数据
- 当指标全为0时，尝试切换到相关指标

### 5.2 分析异常处理
- 当SQL查询失败时，切换到RAG检索
- 当RAG检索无结果时，返回兜底话术
- 当用户问题超出域范围时，返回兜底话术
