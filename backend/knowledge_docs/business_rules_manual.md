# 电商平台业务规则说明手册

## 1. 平台概述

### 1.1 平台定位
- **平台名称**：贵州省电商数据分析平台
- **平台定位**：为贵州省内电商企业和相关部门提供数据分析和决策支持
- **服务对象**：电商企业、政府部门、研究机构、投资者等
- **核心功能**：数据查询、数据分析、报告生成、趋势预测等

### 1.2 业务范围
- **区域范围**：贵州省全境
- **时间范围**：2025年全年
- **数据类型**：销售数据、用户数据、商品数据、行为数据等
- **分析维度**：区域、时间、品类、用户等多个维度

## 2. 数据规则

### 2.1 数据采集规则
- **结构化数据**：从MySQL数据库采集，包括以下表：
  - ads_region_consume_analysis：区域消费分析表
  - ads_product_feature_full：商品特征宽表
  - ads_user_item_interaction_matrix：用户-商品交互矩阵表
  - ads_user_profile_full：用户画像标签表
- **非结构化数据**：从knowledge_docs目录采集，包括：
  - SOP文档
  - 市场报告
  - 经济年鉴
  - 知识文档
  - 业务规则说明
  - 数据分析方法论

### 2.2 数据处理规则
- **数据清洗**：处理缺失值、异常值和重复值
- **数据标准化**：统一数据格式和单位
- **数据转换**：将非结构化数据转换为向量存储
- **数据更新**：定期更新结构化数据，实时监控非结构化数据变更

### 2.3 数据安全规则
- **访问控制**：基于角色的访问控制
- **数据加密**：敏感数据加密存储
- **审计日志**：记录所有数据访问和操作
- **数据脱敏**：对敏感信息进行脱敏处理

## 3. 业务流程规则

### 3.1 数据分析流程
1. **需求分析**：明确分析目标和需求
2. **数据采集**：从结构化和非结构化数据源采集数据
3. **数据预处理**：清洗、转换和标准化数据
4. **数据分析**：使用适当的分析方法和工具进行分析
5. **结果可视化**：生成图表和报告
6. **结论与建议**：基于分析结果提出结论和建议

### 3.2 问题处理流程
1. **问题接收**：接收用户问题
2. **意图识别**：识别用户意图和需求
3. **数据查询**：根据意图查询相关数据
4. **结果分析**：分析查询结果
5. **回答生成**：生成用户回答
6. **质量检查**：检查回答质量和准确性

### 3.3 异常处理流程
1. **异常检测**：检测数据异常和系统异常
2. **异常分类**：对异常进行分类和评估
3. **异常处理**：根据异常类型采取相应的处理措施
4. **异常记录**：记录异常情况和处理结果
5. **异常预防**：分析异常原因，采取预防措施

## 4. 分析规则

### 4.1 指标计算规则
- **销售额**：实际销售的商品金额总和
- **订单数**：成功完成的订单数量
- **客单价**：销售额除以订单数
- **转化率**：购买人数除以访问人数
- **复购率**：重复购买的用户数除以总用户数
- **付费率**：付费用户数除以总用户数
- **销量**：实际销售的商品数量

### 4.2 维度分析规则
- **区域维度**：按城市、省份等地理区域进行分析
- **时间维度**：按年、月、日等时间单位进行分析
- **品类维度**：按商品类别、品牌等进行分析
- **用户维度**：按用户画像、行为特征等进行分析
- **渠道维度**：按销售渠道、推广渠道等进行分析

### 4.3 分析方法规则
- **描述性分析**：使用统计方法描述数据特征
- **诊断性分析**：分析数据异常和问题原因
- **预测性分析**：基于历史数据预测未来趋势
- **规范性分析**：分析最优决策方案

## 5. 回答规则

### 5.1 回答格式规则
- **结构清晰**：使用标题、列表等格式，使回答结构清晰
- **内容准确**：基于数据和事实，确保回答准确无误
- **语言简洁**：使用简洁明了的语言，避免冗长和复杂的表述
- **逻辑连贯**：回答内容逻辑连贯，前后一致
- **专业术语**：使用适当的专业术语，确保表达准确

### 5.2 回答内容规则
- **基于数据**：所有回答必须基于实际数据，避免主观臆断
- **业务相关**：回答内容必须与电商业务相关
- **范围限定**：只回答贵州省电商相关问题，超出范围的问题返回兜底话术
- **数据来源**：明确说明数据来源和时间范围
- **局限性**：如实说明数据和分析的局限性

### 5.3 兜底规则
- **域外问题**：对于超出电商分析范围的问题，返回兜底话术
- **数据不足**：对于数据不足的问题，说明数据覆盖范围并提供替代方案
- **技术问题**：对于技术故障导致的问题，说明原因并提供解决方案
- **歧义问题**：对于存在歧义的问题，向用户澄清并获取更多信息

## 6. 权限规则

### 6.1 用户角色
- **普通用户**：可查询基本数据和分析报告
- **企业用户**：可查询企业相关数据和定制分析报告
- **政府用户**：可查询区域整体数据和政策相关分析
- **管理员**：可管理用户权限、数据和系统设置

### 6.2 权限管理
- **权限分配**：根据用户角色分配相应的权限
- **权限审批**：权限变更需要经过审批流程
- **权限审计**：定期审计权限使用情况
- **权限回收**：当用户角色变更时，及时回收相应权限

## 7. 系统规则

### 7.1 系统架构
- **前端**：Vue 3 + Vite，提供用户界面
- **后端**：Flask + SSE，提供API服务
- **数据库**：MySQL + Chroma，存储结构化和非结构化数据
- **模型**：Ollama本地LLM，提供AI分析能力

### 7.2 系统配置
- **数据库配置**：在config.py中配置MySQL连接信息
- **模型配置**：在config.py中配置LLM模型参数
- **RAG配置**：在config.py中配置RAG检索参数
- **服务器配置**：在server.py中配置Flask服务器参数

### 7.3 系统维护
- **数据备份**：定期备份数据库和向量库
- **系统更新**：定期更新系统和依赖库
- **性能监控**：监控系统性能和响应时间
- **安全检查**：定期进行安全检查和漏洞修复

## 8. 报告规则

### 8.1 报告类型
- **日常报告**：定期生成的常规分析报告
- **专题报告**：针对特定主题的深度分析报告
- **定制报告**：根据用户需求定制的分析报告
- **预警报告**：针对异常情况的预警报告

### 8.2 报告结构
- **标题**：明确报告主题和时间范围
- **摘要**：简要说明报告内容和核心发现
- **目录**：列出报告的主要章节和内容
- **正文**：详细的分析内容和数据
- **图表**：使用图表展示数据和分析结果
- **结论**：基于分析结果得出的结论
- **建议**：基于分析结果提出的建议
- **附录**：相关数据和资料

### 8.3 报告生成流程
1. **需求确认**：确认报告需求和目标
2. **数据收集**：收集相关数据
3. **数据分析**：进行数据分析和处理
4. **报告编写**：编写报告内容
5. **质量检查**：检查报告质量和准确性
6. **报告发布**：发布报告并提供给用户

## 9. 术语定义

### 9.1 业务术语
- **电商**：电子商务，通过互联网进行商品和服务交易的活动
- **GMV**：商品交易总额，包括付款和未付款的订单
- **客单价**：平均每个订单的金额
- **转化率**：从访问到购买的转化比例
- **复购率**：用户再次购买的比例
- **LTV**：用户生命周期价值，用户在生命周期内为平台带来的总价值

### 9.2 技术术语
- **RAG**：检索增强生成，结合检索和生成的AI技术
- **LLM**：大型语言模型，如GPT、Qwen等
- **向量库**：存储文本向量表示的数据库
- **SSE**：服务器发送事件，用于实时数据传输
- **SQL**：结构化查询语言，用于数据库查询
- **API**：应用程序编程接口，用于系统间通信

### 9.3 分析术语
- **维度**：分析的角度，如时间、区域、品类等
- **指标**：衡量业务表现的数值，如销售额、订单数等
- **漏斗分析**：分析用户从访问到购买的转化过程
- **用户画像**：用户的基本特征和行为特征
- **归因分析**：分析影响指标变化的原因
- **预测分析**：基于历史数据预测未来趋势
